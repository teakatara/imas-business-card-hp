# 名刺カルーセル画面 仕様書

## 目的
名刺カードをカルーセル表示し、カードごとの表裏切り替えを行えるページを実装する。
また、名刺カードの下に担当アイドルと好きな曲を表示でき、さらにブランド絞り込みラジオボタンでその内容を絞り込んで表示切替を行うことが出来るページを実装する。

## 画面の考え方
- PCでは前・中央・次の3枚を横並びで表示する。
- モバイルでは中央カードのみを主表示する。
- 絞り込みは単一選択とし、選択中ブランドに応じて表示対象を切り替える。
- ラジオボタンの選択された内容に応じて「担当表示エリア」「好きな曲表示エリア」を同期更新する。

15. 好きな曲表示エリア

## カルーセル仕様
- 表示対象は常に前・中央・次の3枚。
- 前後移動ボタンでカードを切り替える。
- 末尾から次へ進むと先頭へ戻るループ方式とする。
- ループ時も前進と同じ感覚で扱えるよう、表示方向は固定する。

## 裏表スワップ仕様
- 各カードは表と裏を持つ。
- スワップ状態はカードごとに独立して保持する。
- カード移動後も、同一カードの表裏状態は維持する。

## 絞り込み仕様
- 絞り込みは単一選択。
- ALL は全件表示。
- それ以外のブランドを選択すると、該当ブランドの担当アイドルと好きな曲のみを表示する。

  - 単一カードの表裏描画

  - ブランド絞り込みUI
  - 単一選択のラジオ相当UI

## 状態管理
- `selectedBrand`
- `currentIndex`
- 派生値
  - `currentCard`
  - `prevCard`
  - `nextCard`

## データ設計
カードデータには少なくとも以下を含める。
- `id`
- `cardTitle`
- `cardImage`

## 検証項目
- 末尾まで移動した際に逆サイドのカードがループする
- 表裏スワップがカードごとに独立している
- モバイル幅で中央カードのみ表示される
- ビルドが成功する

## 実装メモ
- 1920×1080 の座標は相対配置の目安として扱う。
- 実装はレスポンシブレイアウトを優先する。
- まずは静的データで構成し、将来のデータ拡張に備える。
